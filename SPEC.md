# SysViz - System Visualizer

OSSプロジェクトの動作を3Dビジュアライズして理解するための学習プラットフォーム。

## コンセプト

任意のシステムの内部動作（通信、処理、変換）をThree.jsで3Dアニメーションとして表示する。
見るだけでなく、処理を変更してどう動きが変わるかも確認できる（上級モード、将来実装）。

## 目的

- 対象システムの理解を深める
- 「使えるレベル」の知識を身につける
- 可視化ツールを作る過程自体がReact/Next.js/Three.jsの学習になる

---

## Phase 1: Flask可視化UI

Flaskの「1リクエストがどう処理されるか」を可視化する最初の実装。

### 技術スタック

| 技術 | 役割 |
|---|---|
| Next.js | アプリフレームワーク |
| React Three Fiber | Three.jsのReactラッパー（3D描画） |
| drei | 3Dヘルパーライブラリ（テキスト・カメラ制御等） |

### 画面仕様

| 項目 | 内容 |
|---|---|
| エントリ | スタートボタンを押して再生開始 |
| 操作 | スタート / ストップ / 一時停止 のみ |
| テーマ | クリーン×ミニマル（白背景、落ち着いた色使い） |

### ノード表現

- **情報量**: ノード名 + 入力データ + 出力データ
- **レイアウト**: 3D空間内に階層レイヤーで配置（クライアント層・サーバー層・アプリケーション層）
- **データ移動**: パーティクル（光の玉）がノード間を移動。パーティクルにデータ内容を表示
- **カメラ**: アクティブなパーティクルに自動追従

### Flask 1リクエストのノード構成

```
[ブラウザ] → [WSGIサーバー] → [ルーティング解析] → [ビュー関数] → [レスポンス生成] → [ブラウザ]
```

各ノードの例：

| ノード | 入力 | 出力 |
|---|---|---|
| ブラウザ | ユーザー操作 | HTTPリクエスト (GET /hello) |
| WSGIサーバー | HTTPリクエスト | WSGI environ辞書 |
| ルーティング解析 | environ辞書 | マッチしたview関数 |
| ビュー関数 | URLパラメータ | レスポンスオブジェクト |
| レスポンス生成 | レスポンスオブジェクト | HTTPレスポンス (200 OK) |
| ブラウザ | HTTPレスポンス | 画面表示 |

### 将来の拡張予定

- ノードをクリックで詳細表示（重要な関数の内部処理）
- 上級モード（パラメータ変更・エラー発生の再現）
- 他プロジェクト対応（grpc-go, livekit）
- 汎用化（任意のGitリポジトリを自動解析）
